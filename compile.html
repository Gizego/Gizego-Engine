Compiling the Library
This is different for every platform

Windows MinGW (w/ C++11 support):

The engine was made for GCC. With MinGW you should get everything you need.
The libraries in the "dependencies" folder were compiled for windows so you won't have to build any dependency.

x86:
g++ main.cpp include/glad/glad.c -std=c++11 -static ./dependencies/x86/libglfw3.a -static ./dependencies/x86/OpenAL32.lib -lopengl32 -lgdi32 -o game32.exe

x64:
g++ main.cpp include/glad/glad.c -std=c++11 -static ./dependencies/x64/libglfw3.a -static ./dependencies/x64/OpenAL32.lib -lopengl32 -lgdi32 -o game32.exe

Visual Studio on Windows:

This is untested but you will have to rebuild GLFW (Build guide here) to a MSVC library.
Make a solution file and statically link the libraries in the "dependencies folder" for your targeted CPU architecture.


Linux and Mac GCC (w/ C++11 support):

In this case, you'll have to rebuild all the libraries for your platform.
You'll extract some important information from the BAT file. It won't work directly on your platfrom but it shows you what you should do with GCC.

x86:
g++ main.cpp include/glad/glad.c -std=c++11 -static ./dependencies/x86/libglfw3.a -static ./dependencies/x86/OpenAL32.lib -lopengl32 -o game32.exe

x64:
g++ main.cpp include/glad/glad.c -std=c++11 -static ./dependencies/x64/libglfw3.a -static ./dependencies/x64/OpenAL32.lib -lopengl32 -o game32.exe
